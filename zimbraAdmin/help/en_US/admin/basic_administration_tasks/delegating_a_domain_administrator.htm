Assigning a Domain Administrator to A Domain

The global administrator can designate domain administrators who can create and maintain accounts, aliases, distribution lists, and resources on a specific domain.

Domain administrators must have an account on the domains they administer.

Procedure

  1. Go to Configure>Class of Service  to create a Class of Service (COS) with the features and attributes that the accounts on the domain should be assigned.

  2. Go to  Manage>Accounts, to create a domain administrator account and assign the account to the COS created in step 1.

  3. Go to Configure>Domains, and select the domain to be assigned a domain administrator account.

  4. Click the gear icon and select Edit.

  5. In the General Information page, Default Class of Service field, type this COS name. All accounts the domain administrator creates on the domain after the COS is assigned are assigned this COS. The global administrator can change an account's features and attributes either by assigning a different COS to the account or by modifying the account Features, Preferences and Advance tabs.

    • If you do not assign a COS to the domain, the domain uses the default COS for that domain.
