Setting Account Limits for Domains

You can limit the number of accounts that can be provisioned on a domain.  The maximum number of accounts allowed on a domain can be set when the domain is created.  You can also edit the domain's Account Limits page to set up account limits or to change them.

When the account limit is reached, if you provision a new account, when you try to save the account, a message displays showing that the account limit has been reached, and the account is not created. Resource accounts are not counted against the limit you set.

When multiple Classes of Service (COS) are available,  you can select which classes of service can be configured and how many accounts on the domain can be assigned to the COS.

The number of account types assigned is tracked. You can see the number assigned/number remaining from any account's General Information page.

To set or change the limit on a domain and set the COS limits

  1. Go to Configure > Domains and select the domain to edit.

  2. Open the Account Limits page and enter the maximum number  of accounts for the domain in the Maximum accounts for this domain field.

  3. If you want to configure specific COS for the domain, click Add, otherwise go to Step 6.

  4. In the Add maximum accounts dialog Class of Service field

    • Enter the first letters of the COS and then select the COS from the list that displays.

    • In the Limit field, enter the number of accounts that can be created.

    • Click OK.

    • The COS name that displays in the Account Type is taken from the COS Description field. If that field is blank, the COS name is displayed.

  5. Click Add to add additional COSs as described in step 4.

  6. Click Save. Click Close.

  7. Click the Browser Refresh button to update the information in the administration console.
