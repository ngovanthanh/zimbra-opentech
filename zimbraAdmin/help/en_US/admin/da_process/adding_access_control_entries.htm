Adding Access Control Entries

An access control entry (ACE) is the specific access granted on a target. You can add ACEs when you grant rights to a delegated administrator and you can add ACE's directly to the target.

  1. Click Add to add an ACE. Select the Target Type from the menu.

  2. Enter the target's name. The name must be the same as configured in Zimbra Collaboration Server. For example, domain_name@example.com.

  3. Select the type of right to grant, either System Defined Right or an individual Attribute Right.

If you select System Defined Right, you must enter the name of the right associated with that target.  For a list of rights, see the Rights folder in the Overview Pane Configuration section.

If you select Attribute Right, you must select whether the right is read-only or write. The attribute right target type associated with the target type you selected are listed.

  1. In the Right Name field, enter the attribute associated with the right.

  2. Select whether this is a positive or negative rights. When you select Is Positive Right, if this administrator can grant the same right to other administrators,  check Can grant the right to other admins.

  3. Click Add and More to add this right and open the Add ACE window again to add another right. Click Add and Finish to add this right and finish the configuration.

When an ACE is created, the ACE includes the following information: Zimbra ID of the grantee account, type of grantee, either user (usr) or group distribution list (grp), and the allowed or denied right. (+ or -).
