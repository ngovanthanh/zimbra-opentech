Account Provisioning

All accounts that had the status of No such account are shown in the list box.

  1. Before creating these accounts, select the class of service that these accounts will be assigned.

As a minimum two entries are listed. Default refers to the "default" COS that is automatically created by Zimbra. None in the COS field is configured to refer to the domain's default COS, if one has been created.  Select None to provision the accounts with the domain COS that you have configured as the default. If None is selected and the domain does not have a default COS, the Zimbra "default" COS is used.  

    1. If you recently created a COS and it is not listed, click Refresh COS. Any new classes of service will be added to the list.

  1. Enter the default password for the accounts. This is the password for every account that is created. Note: If a password is not specified at this time, refer to the migration guide for different options.

  2. To provision the listed accounts, click Next. The accounts are created on the Zimbra server. At this point, the provisioning of email accounts on the Zimbra Server is complete.

    • Note: If you do not want to create any of the accounts from this list, check Do not provision any users, and click Next. No accounts are created.

  3. Before proceeding, you should change the mail relay information for these users to point to the Zimbra server instead of the Exchange server. Any new mail will be sent to the Zimbra mailboxes.
