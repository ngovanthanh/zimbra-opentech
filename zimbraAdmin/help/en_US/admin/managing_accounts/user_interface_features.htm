Features to Enable

The Zimbra Web Client (ZWC) interface features that can be enabled by either COS or Account are listed below. In addition to these features, Preferences and Advanced features can be enabled to enhance how some of these features work.

Feature

Description
Major Features

Mail

Users can receive, create, and manage email messages.

Contacts

Users can create their own contacts lists. When this is enabled, a Contacts and an Emailed Contacts address book are created in the user's Address Book.

Calendar

Users can create Calendars and use the scheduling tool to maintain their calendars.

Tasks

Users can create and organize tasks.

Briefcase

Users can upload files and create new documents and can access the files from ZWC.

Preferences

Users can change their ZWC preferences for mail, calendar, address book, and other features.
General Features

Tagging

Users can create tags and apply tags to items in ZWC. Users can apply multiple tags to a single item to classify it under several categories simultaneously.   

Sharing

Users can share their email Inbox, address books, calendars, tasks, and briefcase folders with others, either those in their organization, external guests, or the public.

Change Password

Checking this requires users to change their log in password the first time they log in after the administrator changes the password. This setting is used if you configure internal authentication.

Change UI Themes

Users can change the background colors in their view of Zimbra Web Client.  The themes that are available are configured in the COS Themes tab.

Manage Zimlets

Users can enable or disable Zimlets from their accounts.

HTML compose

Users to compose rich emails that can contain different fonts, colors, and other style choices.

Global Address List access

Users can access the Global Address List.

MAPI (Microsoft Outlook) Connector

Users can use the Zimbra Connector for Outlook

EWS (Exchange) client access

Users can use Exchange Web Services (EWS).

Touch Client (Mobile Web App)

Users can use the Touch Client.

Offline support for Advanced (Ajax) client

Users can use Offline Mode.

Autocomplete from GAL

As users type in the address field when composing an email, suggested addresses from GAL that match the text are displayed. Users can select an address rather than type the complete address.

Import/Export

Users can import and export address books, calendars and other item folders

Dumpster folder

Users can recover items that they have deleted from their Trash folders. When this is enabled, users can right-click on the Trash folder and select Recover Deleted Items to display messages deleted up to 30 days before.

Can purge dumpster folder

If Dumpster folder is enabled, this is displayed.
Mail Features

Message priority

Enables the message priority feature so that users can set a priority indicator for messages they send, Normal, Low, or High. This is an alert to the recipient. The message is not sent out faster or slower based on the message priority.

Flagging

Users can flag messages. Flags can be used to indicate an action-needed item or to otherwise distinguish the message or conversation.  

IMAP access

Enables users to use third party mail applications, such as Thunderbird or Outlook to access their mailbox using the IMAP protocol.

POP3 access

Post Office Protocol. Enable users to use third party mail applications, such as Thunderbird or Outlook to access their mailbox using the POP3 protocol.

External IMAP access

Users can set up to retrieve their IMAP accounts' email messages directly in Zimbra Web Client. When they retrieve their IMAP email messages, the messages and attachments are saved on the Zimbra server.

External POP access

Users can set up to retrieve their POP accounts' email messages directly in Zimbra Web Client. When they retrieve their POP email messages, the messages and attachments are saved on the Zimbra server.

Allow the user to specify a forwarding address

Users can create a forwarding address for their mail. You can specify a default forwarding address or users can add the address from their accounts.

Mail send later

When this is enabled, users can right-click the Send button, and select Send Later to send a message at a later time. They configure a date and time to send an email message and the message is saved in the Draft folder until the time.

Conversations

Messages can be displayed grouped into conversation or as a message list.  Conversations group messages by subject. If this feature is turned on, conversation view is the default. Users can change the default from their mailbox toolbar.

Mail Filters

Users can define a set of rules and corresponding actions to apply to incoming and outgoing  mail and calendar invites. When an incoming email message matches the conditions of a filter rule, the corresponding actions associated with that rule are applied.

Out of office reply

Users can create an email message that automatically replies to incoming messages.  This is commonly used as a vacation message. By default, a message is sent to each recipient only once every seven days, regardless of how many messages that person sends to the address during that week. Users can set the start and stop dates for sending their message.

New mail notification

When this is checked, users can specify an email address that receives notification that there is email in the Zimbra mailbox.  In the preferences section, you can enable whether the New mail notification is on or off and set the address.

Persona

Users can create multiple mail identities, called personas, which can be applied to email messages they send. For each identity, they set up an email address and name that appears in the From header of messages they send. They can also set up their accounts to receive and reply to external  POP and IMAP accounts from their ZWC account.

Enable read receipts

When Read Receipts is enabled, users can flag a message to request a read receipt, and recipients of the message can set up preferences for how to handle the request: never send a read receipt, always send a read receipt, or ask before sending a read receipt.

Calendar Features

Group calendar

Group calendar must be enabled to have full Calendar functionality to schedule appointments and share a calendar. When Group calendar is disabled, Calendar becomes a personal calendar only. The user can create their appointments, but cannot schedule meetings or share calendars with others.

SMS reminders

Users can set up SMS reminders for their calendar and task items.
Search Features

Advanced search

Users can build detailed search request that include looking up mail by date, domain, flag, object, size, attachment, and time.     

Saved searches

Users can save a search query they create. Saved searches are listed in ZWC Overview pane for easy access.

Initial search preference

Users can select which of their folders to display when they log on. The default folder is Inbox. The folder is automatically searched if no other folder is specified.

S/MIME Features (Secure/Multipurpose Internet Mail Extensions)

Enable S/MIME

Users can send and receive encrypted email messages if they have a PKI certificate and private key suitable for S/MIME. If S/MIME certificates are stored on an external LDAP server, configure the LDAP URL as either a global or domain setting. See Configuring_S_MIME_Settings.htm
